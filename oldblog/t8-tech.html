What is "Ruby on Rails"?

You see and hear this a lot once you've entered the coding world. You'll see job descriptions specifically seeking out "RoR developer" and coding on "Rails." But what is it? Of course by now you know that Ruby is a programming language. And with each language they have their own personalities. And maybe you've heard this quote by Bill Gates:

"I choose a lazy person to do a hard job. Because a lazy person will find an easy way to do it."

Why does this apply to programming? Because that means these "lazy people" are most likely to be the ones who will find a solution that is the quickest and simplest. Code that is D.R.Y. (Don't Repeat Yourself). And it's based off of this mentality that Rails was born. Rails is a framework created to assist the programmer in coding better and faster. It's a framework that is instilled with the ideal that you don't need repetitive code and is preprogrammed with the specific personality/style of ruby coding.

I think I'll have more to share with you on this once I'm actually using this framework, but for now this is just a general definition of what Rails is. So that's all folks for this time. See you all in the next post!
