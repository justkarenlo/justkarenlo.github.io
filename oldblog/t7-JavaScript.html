The one with the Ruby and JavaScript comparison


11/02/2014 :: Phase 0 Week 7

When I first began this journey into this amazing coding world, I was told that once you learn a language, and I mean really learn it, it's really quite simple when you want to pick up another language. I can see that now. This week, my cohort family and I were introduced to JavaScript. And boy, it was a doozy. As soon as I began reading from our JavaScript textbook, I realized immediately that I need to take notes. And I did. I began a table that showed the differences between JavaScript and Ruby. What I noticed most importantly, was that yes there are differences between the two, but ultimately, it felt like dealing with a 'sibling' of Ruby. You had methods that do similar things, but the syntax is the only thing that is different (at least to the naked eye of a beginning programmer). I created the following table to show the similarities and differences for myself; but heck, why not share this 'nugget' of knowledge with you all? It's not a complete list by all means, but definitely a quick reference list that I, myself, will definitely be looking to every now and again.

As you can see, there are a lot of actions to objects that match up between Ruby and JavaScript; however, the syntax is quite different. I hope that this list will give you an idea of how you can solve JavaScript problems if you're more familiar with Ruby and vice versa.

Of course syntax isn't the only thing that's different between the two languages. The way each handles and defines an object is quite characteristic and is why one language can be more powerful than another in certain aspects.

Ruby JavaScript
An object is:
  • an instance of a class (a class which has been defined by attributes and methods/functions)
    • this is why it you can use 'self' only after you've created a Class and defined an instance
    • this is also why arrays, hashes, strings, numbers, etc. are considered objects; they are instances of those corresponding classes (ex. Array.new, Hash.new, String.new)
An object is:
  • everything.
  • a function (this is why it can call upon itself using 'this'
  • more powerful in JavaScript than in Ruby because of its ability to house, add, and modify methods/functions outside of the object itself
    • this can be accomplished without the use of a class/prototype/template (I view these three as synonymous)
    • Example:
var monica = {
    name: "Monica";
    career: "chef";
    spouse: "Chandler";
    apartment: "dirty";
    // here it comes...
    clean: function () {
        this.apartment = "clean";
        // you can also use monica.apartment = "clean";
    }
    // ladies and gentlemen, we have ourselves an object
      //within an object! Do you see it?
  }

From the example above you can see how JavaScript has an increased functionality when it comes to its objects. The first object is: the defined monica = {} and the object within the monica object is: clean function. But HOLD THE PHONE, doesn't the object monica, look just like a hash in Ruby?

            Ruby hash
            hash_name = { key => value }

            JavaScript object
            var object_name = { property : value }

As mentioned in the table above, only JavaScript allows you to do more with the data you provide. JavaScript allows you to add, modify, and remove these objects just as easily outside of the object. 

              Adding a NEW object
              monica.play_game = function {
                  console.log("Monica throws plate");
            }
              Removing an object
              delete monica.clean;

              Altering object (similar to adding)
              monica.play_game = function {
                  console.log("Monica loses apartment");
            }

As you can see, we could never do any of this to our basic hash in Ruby. In this way, JavaScript is more powerful due to its object-oriented driven programming. I hope I helped you ease into the understanding of the differences and similarities between Ruby and JavaScript. I know for darn sure it's confusing at first, but with practice (as with anything in life) we'll get there.
