"Know-How"

Padding/Border/Margin

So as a fellow web dev student or just someone who's interesting in coding, the first wall you hit is most likely, what's the difference between margin, border and padding? I have to say the best technique I've learned to assist me throughout my coding challenges are this: in the stylesheet (.css file) use the selector "*" to select all elements of a site and create a border (ex. 2px solid black (or white for my site)). once you've done that you'll be able to see all the borders around the content and focus on just the spacings around everything.

Let's focus on the content and work our way out through each layer. Content is the actual "stuff" you have in your div, p, table, td, etc. so most likely, the TEXT. The area/space around the content and border is called a padding. And then the zone outside of the border is your margin.

Just think of a shipping box.

Content is what you ordered from Amazon. Padding is the styrofoam balls or bubble wrap. Border is the cardboard box that the item was shipped in. And finally, the margin is the space between your box and other boxes that the UPS/USPS shippers give when they throw all their packages into the trunk just before their deliveries. Helpful? Certainly helped bite off some of the edge when I think of it that way. Almost reminds me of the time I learned about the mitochondria and golgi bodies...ANYWAYS.

classes & ids

Moving on...one of greatest tools to use in html is using classes and ids. The best practice to use these are, in my opinion, is to plan ahead. If you know you'll be using some of the same characteristics/themes (design-wise) you'll want to use class. IDs are so specific by nature, that they work great when you know there's only one part of your site you want to look the way it looks.

relative, absolute, and fixed positioning

AKA. hell on earth. jk... -ish. You have no idea (or actually, maybe you do) know how much I have struggled with this:

fixed positioning is when the selected item is positioned in association to the browser. This positioning does not budge, so if you're scrolling through a page, it will be nailed down to the same location of that page wherever you go!

relative positioning is when the selected item is positioned in association to its original/default/static position; not the browser. So if you have multiple elements, you'll have to remember where the div/table/etc. placement was original and move (top/left) accordingly.

absolute positioning is an interesting one. so first of all, its positioning is in association to whatever the parent's position is, IF that parents position type is NOT static. So example, I have a div with an id "parent" and a div within the "parent" div called "child." If the child's position is set to absolute, it will look to see if it's parents position is static or other. If the parent's position is static, the child will move in association to the browser. If the parent's position is other (ie. absolute/fixed/relative) then it will move in association to the position the parent is currently in.

Wheww lots to take in, but I hope that was clear. I had someone just run this through with me..again.. and I think I'm finally getting it...-ish.

inline vs inline-block

Last of our topics for tonight, SO the display properties inline and inline-block are relatively easy to understand. When a display is set to inline it will condense the divs/tables to the bare minimum size based on the content (content size is left untouched) and also place them...in line or in a row fashion. Now a inline-block display will do the same but instead it will expand the padding around the content to fit the full potential of a block and then place each into a row.
